Coconut: Chain of Continuous Thought

Coconut takes a very simple but unusual idea and pushes it all the way through training and inference: do not force every reasoning step through language. Instead, let the model keep part of the reasoning process inside its hidden state, feed that hidden state back as the next input embedding, and only decode to words when it is time to speak again.

That sounds like a small wiring change, but it changes the shape of reasoning. In chain-of-thought, the model must verbalize every intermediate step, even when those words are only there to keep the computation moving. Coconut asks a different question: what if the useful part of the reasoning is the internal state itself, not the token that happens to express it?

What to notice here is the break from token-by-token commitment. CoT walks forward in words. Coconut turns the last hidden state into a reasoning state and loops it back in. That means the model can keep exploring before it chooses a final verbal answer.

Why the paper thinks language is the wrong bottleneck

The paper’s starting point is a practical complaint about chain-of-thought. Not every step in a reasoning chain deserves the same amount of computation. Many tokens are just there for fluency. They make the chain readable, but they do not help the actual inference very much. Other tokens are crucial decision points where the model has to plan, search, or choose between alternatives.

That mismatch is the whole motivation for Coconut. A language model already computes rich hidden states while it processes text. So instead of forcing those states to be turned into words immediately, Coconut keeps them in latent form for a few steps. The reasoning process becomes a hidden trajectory, and the model only comes back to language when it has something worth decoding.

The paper is careful not to oversell this as magic. Coconut is still a post-training method. It does not remove the need for supervision. What it does is move the intermediate reasoning work into a space that is better suited to search and less constrained by natural-language surface form.

The core mechanism: one hidden state becomes the next thought

The method alternates between two modes.

In language mode, the model behaves normally. It reads token embeddings, computes hidden states, and predicts the next token.

In latent mode, it stops using the embedding of a word token and instead feeds the previous step’s last hidden state back in as the next input embedding. Coconut calls that hidden vector a continuous thought. The paper uses special <bot> and <eot> tokens to mark the beginning and end of that latent segment.

So the sequence looks like this:

  1. read the question in ordinary language,
  2. enter latent mode at <bot>,
  3. reuse the last hidden state as the next embedding,
  4. repeat that for several latent steps,
  5. exit latent mode at <eot>,
  6. decode back into language and produce the answer.

This is the key design choice in the paper. The model is not learning a new architecture from scratch. It is reusing the transformer’s own internal state as a reasoning token.

The important detail is that the last hidden state is normalized before being fed back, so it is stable enough to serve as an input embedding. That keeps the loop numerically practical.

How the training curriculum teaches latent reasoning

The hardest part is not the inference trick. It is teaching the model to use it.

If you start with raw answers only, the paper finds that the latent space does not naturally acquire useful reasoning structure. The model tends to collapse toward no-CoT behavior. In other words, just asking a model to “think silently” is not enough. It needs a bridge from ordinary reasoning traces to latent reasoning.

Coconut builds that bridge with a staged curriculum.

The picture above is the training story in one glance. The model begins with standard CoT supervision. Then, stage by stage, the paper replaces early language reasoning steps with continuous thoughts. If one language step is replaced by c latent steps, the model has to learn to preserve the reasoning state across a longer hidden trajectory before it returns to text.

That curriculum matters because it gives the model a gentle path from familiar language reasoning to latent reasoning. The paper is not trying to make the model invent latent thoughts from scratch. It is transferring the structure of CoT into hidden space in progressively harder stages.

The loss is straightforward: cross-entropy on the remaining text tokens, while masking the question tokens and latent spans. So the model is not directly punished or rewarded for the latent vectors themselves. Instead, it is trained so that those vectors help predict the later visible tokens correctly.

That design choice is subtle but important. The latent thought is not asked to compress a removed sentence in a human-readable way. It is asked to be useful for predicting what comes next. That gives the model freedom to learn a reasoning representation that may be more efficient than language.

What inference looks like

At inference time, Coconut behaves like decoding with a hidden detour.

The model starts from the question, enters latent mode after <bot>, and runs for a fixed number of continuous thoughts. Then it exits latent mode and decodes the answer in language. For simplicity, the paper often uses a fixed latent length rather than having the model decide dynamically when to stop.

That fixed-length choice is practical. It avoids adding a termination policy before the latent reasoning idea itself has been validated. The paper also notes that a classifier-based stopping rule is possible, but the fixed-length version is enough for the experiments.

One useful way to think about this is that Coconut gives the model an internal scratchpad, except the scratchpad is not text. It is a chain of hidden vectors.

Why latent reasoning can behave like search

The most interesting part of the paper is what Coconut seems to do once the model is allowed to reason in latent space.

On planning-heavy tasks, Coconut often behaves less like greedy token generation and more like a search process. The paper’s interpretation is that a continuous thought can carry several plausible next steps at once. Instead of collapsing immediately onto one word path, the latent state can retain a superposition of options and refine them later.

That is why the authors describe the behavior as breadth-first-search-like rather than purely greedy. The model can postpone a hard choice, keep multiple candidate paths alive, and then eliminate bad ones later.

This case study shows the difference clearly. CoT can get trapped and start inventing unsupported transitions. Coconut with a small number of latent thoughts does better, and with more latent thought steps it reaches the correct path. The important thing to notice is not just the final answer, but the way premature commitment is avoided.

The paper makes that intuition more explicit by probing the hidden state as if it were a value function over candidate next nodes. The latent state assigns scores to possible continuations, and those scores change as reasoning proceeds. Early steps keep more options alive. Later steps become more selective. That is exactly the kind of transition you would expect from a search process: broad exploration first, then narrowing.

Why the curriculum matters for this search behavior

The paper’s strongest negative result is also one of its most informative ones. If Coconut is trained without the staged curriculum, latent reasoning performs poorly. The model does not reliably learn the intended latent structure.

That tells you something important about the method. The continuous thought is not a free lunch. It is a new reasoning channel, but it needs supervision to become meaningful. The curriculum is the mechanism that transfers the structure of language-based reasoning into latent space one stage at a time.

The paper also compares Coconut with variants that replace latent thoughts with ordinary <pause> tokens or remove the thought mechanism altogether. Those variants help a bit in some settings, but they do not reproduce the same behavior as true continuous thoughts. The lesson is that the hidden vector itself is doing work, not just the extra compute budget.

The efficiency story

Coconut is not only about accuracy. It is also about doing reasoning with fewer visible tokens.

On the paper’s planning-heavy logical tasks, Coconut achieves a much better accuracy-efficiency tradeoff than chain-of-thought. On ProsQA, the headline result is 97.0% accuracy with 14.2 tokens, compared with 77.5% accuracy and 49.4 tokens for CoT. On ProntoQA, it reaches 99.8% accuracy with only 9.0 tokens. The clock-time measurements tell the same story: Coconut is substantially faster than CoT on those tasks.

What matters in this figure is the tradeoff curve. As the number of continuous thoughts increases, Coconut can spend more internal compute before it commits to language. That generally improves reasoning on harder tasks. But the method does not scale without cost: latent steps are sequential, so training and inference are not trivially parallelizable.

That is a real limitation. Coconut reduces visible token usage, but it does not eliminate serial computation. It moves the work from words into hidden-state updates, which are still done step by step.

Where the method works best

The paper’s results are strongest on tasks that need search, planning, and delayed commitment.

That is why Coconut shines on ProsQA and ProntoQA. Those datasets reward the ability to keep several possibilities in play and avoid getting stuck too early. Coconut’s latent mode seems to help with exactly that.

On GSM8k, the gains are smaller. The paper still shows that continuous thoughts can improve over no-CoT, but they do not beat the strongest language-chain baselines by the same margin. That is an important nuance. Coconut is not universally better than language reasoning. It is especially attractive when language itself is a poor medium for the intermediate computation.

Another limitation is training complexity. The staged curriculum is effective, but it is also a hand-designed schedule. The paper explicitly says that learning latent reasoning without CoT supervision did not work well, and that more scalable training strategies remain open.

A simple way to remember Coconut

The easiest mental model is this:

the model writes some thoughts in words, then stops speaking and keeps thinking in hidden vectors, then speaks again when it is ready to answer.

Or even shorter:

language is for the final answer; hidden state is for the search.

That is the core contribution of Coconut. It shows that a transformer can treat its own last hidden state as a continuous reasoning step, and that a curriculum can teach that loop to behave like useful search rather than a random latent detour. The paper’s broader message is not that language should disappear. It is that some reasoning gets better when the model is allowed to leave language for a while and only return when it has narrowed the space enough to speak decisively.
