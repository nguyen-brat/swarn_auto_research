DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning

DeepSeek-R1 is easiest to understand as a training recipe, not just a model. The paper asks a sharp question: if we already know whether many reasoning answers are correct, can we teach a language model to reason mainly by rewarding good outcomes, instead of painstakingly labeling every intermediate step?

The answer is yes, but with an important twist. Pure reinforcement learning can uncover strong reasoning behavior, yet the raw result is often messy to read and sometimes mixes languages. So the final system is not just “RL harder.” It is a staged pipeline:

  1. start with pure RL on the base model,
  2. add a small cold-start SFT seed so the model begins with readable reasoning traces,
  3. run reasoning-focused RL,
  4. use rejection sampling to build cleaner SFT data,
  5. run one more RL stage for all scenarios,
  6. distill the learned reasoning patterns into smaller dense models.

That is the whole story. The rest of the paper is about why this sequence works.

The pipeline at a glance

The cleanest way to remember the method is to picture it as a funnel. A strong base model is pushed through reward-driven training, then the useful behavior is cleaned up, expanded, and compressed.

What to notice in the figure is the rhythm of the process. It does not jump straight from base model to polished reasoning assistant. It alternates between:

  • exploration, where RL lets the model discover better reasoning behavior,
  • correction, where a small amount of supervised data makes that behavior readable and reusable,
  • compression, where distillation transfers the pattern into smaller models.

That alternating pattern is the key design choice. DeepSeek-R1 is not trying to prove that one training method alone is enough. It is showing that different post-training stages can do different jobs.

Why pure RL is interesting in the first place

Most LLM training recipes lean heavily on supervised examples. That works well, but it also creates a dependence on hand-built traces. DeepSeek-R1-Zero asks whether reasoning can emerge from reward alone.

The base model is trained with GRPO, which is a group-relative policy optimization method. You do not need the formal objective to understand the intuition. For each prompt, the model samples a small group of candidate answers. Those answers are scored. The model then learns from the relative quality inside that group instead of relying on a separate critic model.

That matters for two reasons:

  • it avoids the cost of training a critic the same size as the policy,
  • it lets the system learn from correctness signals more directly.

In plain language, the model is not being told exactly how to reason. It is being told which solutions work better, and it has to discover the reasoning style that gets rewarded.

The reward signal is the steering wheel

DeepSeek-R1-Zero uses rule-based rewards, not a learned neural reward model. The paper’s logic is practical: if the answer is deterministic, you can often check correctness automatically. Math problems can be verified against the final boxed answer. Code problems can be run against test cases. A format reward can also check whether the model actually puts its thinking inside the expected <think> tags.

This combination is simple, but it is exactly what makes the method scale:

  • correctness reward pushes toward the right answer,
  • format reward pushes toward a stable reasoning template,
  • neither reward requires a separate reward model that might be hacked or retrained later.

That does not mean the model is supervised step by step. It is still learning by trial and error. The difference is that the trial-and-error loop is anchored by verifiable signals instead of subjective labels.

What the base model learns when nobody tells it how to think

The striking part of DeepSeek-R1-Zero is that reasoning behavior appears on its own during RL. The model starts to generate longer chains of thought, self-check its steps, and sometimes revisit earlier assumptions.

The paper highlights an especially memorable “aha moment,” where the model catches itself, says in effect that it should rethink the problem, and then restarts the derivation more carefully. That is not just a cute anecdote. It shows the policy is learning a useful meta-behavior: spend more thinking time when the first pass seems shaky.

You can see that effect in the training curve.

The important thing to notice here is not only that performance rises. It rises as the model learns to use more test-time computation. In other words, RL is not merely nudging final answers. It is shaping the model’s internal habit of reasoning longer when the problem demands it.

That is the core claim of DeepSeek-R1-Zero: if the task has a reliable reward signal, large-scale RL can discover real reasoning behavior without a supervised CoT bootstrap.

Why the zero-shot version still was not enough

DeepSeek-R1-Zero is impressive, but it is not the final answer. The paper is very explicit about its weaknesses.

The biggest issue is readability. The model can produce long reasoning traces, but they may be hard to read, poorly formatted, or mixed across languages. That is a problem if you want a model that users can actually follow.

So DeepSeek-R1 adds a small amount of cold-start supervised data before the RL stages.

This sounds minor, but it changes the whole feel of the system. Instead of asking the model to invent both the reasoning strategy and the presentation style from scratch, the cold-start phase gives it a readable template to imitate.

The cold start is not about teaching knowledge

The cold-start data is small compared with typical supervised pretraining recipes. Its purpose is narrow: make the reasoning traces legible and stabilize the beginning of RL.

The paper describes a response pattern that includes a reasoning process and then a summary at the end. That is a useful design because it gives the model a stable output shape:

<reasoning_process> ... <summary> ...

Why is that valuable?

  • it makes the chain of thought easier to read,
  • it gives the model a clean place to restate the answer,
  • it reduces the chance that RL drifts into chaotic formatting at the start.

This is one of the paper’s most practical lessons. A tiny supervised seed can make the model much easier to train, even if the real capability boost still comes from RL.

The reasoning stage tightens the behavior

After the cold start, DeepSeek-R1 runs another RL stage focused on reasoning-heavy tasks like math, logic, science, and coding.

One extra detail matters here: the model can still mix languages in its chain of thought, especially when prompts are multilingual. To reduce that, the paper adds a language-consistency reward based on the proportion of target-language words in the reasoning trace.

This is a good example of the paper’s overall philosophy. It does not try to solve every problem with one generic reward. Instead, it adds a targeted signal for a known failure mode.

There is a tradeoff, though. The paper notes that this alignment slightly hurts raw performance, but it improves readability and user experience. That is the kind of decision that shapes the final model: not just “what scores highest,” but “what reasoning behavior do we actually want to keep.”

Rejection sampling turns the RL checkpoint into better supervised data

Once the reasoning RL stage converges, the model is no longer just a solver. It is also a data generator.

The paper uses rejection sampling to create a better SFT dataset from the RL checkpoint. The idea is simple: sample several candidate reasoning traces for a prompt, keep the correct ones, and discard the messy ones. That gives you a cleaner supervised corpus than the original cold-start seed.

This stage does two things at once:

  • it captures the better reasoning style the model discovered during RL,
  • it filters out the outputs that are correct but hard to use as teaching examples.

The dataset is broadened as well. Reasoning prompts are kept, but the paper also adds non-reasoning data such as writing, factual QA, self-cognition, and translation. That is important because DeepSeek-R1 is not supposed to become a math-only specialist. It still needs to behave like a general assistant.

This is where the pipeline becomes more than a reasoning engine. It becomes a general post-training recipe.

The final RL stage is about all scenarios, not just puzzles

After the SFT refresh, the model goes through another RL stage. This one uses a mix of reward signals and prompt types so the model can improve helpfulness and harmlessness alongside reasoning.

The paper separates the reward setup by task type:

  • for reasoning tasks, it stays close to the rule-based reward style from DeepSeek-R1-Zero,
  • for general tasks, it uses preference-style reward models,
  • for helpfulness, the model is judged mainly on the final summary,
  • for harmlessness, the whole response is considered.

That split is easy to miss, but it is important. The paper does not want the reasoning trace to dominate every evaluation. A long chain of thought is useful, but the final assistant still has to answer well in ordinary conversations.

So the last RL stage is where DeepSeek-R1 stops being a special-purpose reasoning experiment and becomes a more balanced assistant.

Distillation is how the paper packages the discovery

The final move is distillation. Once DeepSeek-R1 has learned the useful reasoning patterns, the paper fine-tunes smaller open-source dense models on the generated reasoning data.

This is a crucial part of the story. The paper is not saying that everyone should run huge RL jobs forever. In fact, it argues the opposite: large-scale RL is powerful, but expensive. If a stronger model has already discovered a good reasoning policy, it can act as a teacher.

That is why the distilled models matter. They show that the reasoning style can transfer to smaller checkpoints more efficiently than trying to discover the same behavior from scratch with RL on the small model.

The practical takeaway is simple:

  • RL discovers the behavior,
  • distillation spreads it.

That division of labor is one of the most useful ideas in the paper.

What this method is really teaching

DeepSeek-R1 is not just a better benchmark runner. It is a demonstration of a training principle:

if the task has verifiable outcomes, reward-driven post-training can produce reasoning behavior even without expensive step-level supervision, and a small amount of supervised scaffolding can make that behavior readable and reusable.

That principle explains the whole pipeline.

  • Pure RL is the exploration engine.
  • Cold-start SFT makes the output human-friendly.
  • Rejection sampling turns the model’s own good behavior into cleaner teaching data.
  • Final RL broadens the assistant beyond reasoning-only prompts.
  • Distillation compresses the result into smaller models.

What to remember

If you remember only one thing, remember this:

DeepSeek-R1 is a staged recipe for turning reward signals into reasoning behavior.

The base model first learns to improve from correctness and format rewards alone. Then a small supervised seed makes the reasoning readable. Then the model cleans up its own traces through rejection sampling and gets one more RL pass for general assistant behavior. Finally, the discovered reasoning style is distilled into smaller models.

That is why the paper matters. It shows that reasoning can emerge from reward-driven self-improvement, but also that the strongest version of the idea is not a single training trick. It is a pipeline.
