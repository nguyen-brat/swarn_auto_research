Kahneman-Tversky Optimization: Aligning Language Models with Human Feedback

KTO is a different way to think about alignment. Instead of asking the model to learn from a pairwise preference like “this answer is better than that one,” it asks a simpler question: is this output desirable or undesirable for this prompt? The paper’s claim is that, if you keep the right reference point and the right gain/loss shape, that weaker signal can still drive strong post-training.

That sounds modest at first, but the idea is surprisingly deep. The paper is not just saying binary feedback is cheaper. It is saying the loss itself matters. If the objective mirrors the way humans actually judge gains and losses, then the model can learn a lot from coarse labels.

The core intuition

The easiest way to remember KTO is to think of it as “score every output against a baseline, then push it toward or away from that baseline depending on whether it was judged good.”

That baseline is not arbitrary. KTO builds a KL-based reference point from the current model and the reference model, then measures how far an output has moved relative to that point. A desirable output should end up on the gain side of the curve; an undesirable output should end up on the loss side.

What to notice in the figure is the shape story. KTO sits inside the paper’s broader “human-aware losses” view, where the loss is not just a numerical trick but a value function with a reference point. The important part is the asymmetry: losses can be treated more sharply than gains, and the objective can saturate instead of pushing forever.

What problem KTO is trying to solve

Preference data is powerful, but it is also expensive. It requires comparing two outputs for the same prompt, and that structure is not always available in practice. KTO asks whether we can keep most of the benefit of preference-based alignment while only using a binary label.

The practical motivation is straightforward. If you can collect “good” and “bad” outputs more easily than pairs, you can collect more data. But the paper’s point is stronger than that. It argues that a good alignment objective can recover a lot of the value of preference learning even when the feedback is coarse, as long as the loss encodes the right inductive bias.

So the pipeline is:

  1. start with a prompt-output example,
  2. label the output desirable or undesirable,
  3. compare it to a KL-based reference point,
  4. apply a logistic value function with separate weights for good and bad examples,
  5. update the policy directly.

That is the whole mechanism in miniature.

From policy ratios to implied reward

KTO starts by reusing the same basic quantity that shows up in many alignment methods: the log ratio between the current policy and the reference policy. If the current model assigns much more probability to an output than the reference model does, the output has become more “rewarded” under the aligned policy.

The paper writes this as an implied reward:

rθ(x,y)=log⁡πθ(y∣x)πref(y∣x)r_{\theta}(x,y)=\log\frac{\pi_{\theta}(y\mid x)}{\pi_{\text{ref}}(y\mid x)}rθ​(x,y)=logπref​(y∣x)πθ​(y∣x)​

This is the quantity KTO will judge. A positive shift means the model likes the output more than the reference did; a negative shift means the model has moved away from it.

But KTO does not stop at the raw log ratio. It compares that reward to a reference point, which is estimated from the current microbatch as a KL-based baseline. That is the part that makes the method feel prospect-theoretic: the model is not just looking at absolute score, but at score relative to a baseline.

The value function then splits by label:

v(x,y)={λD σ ⁣(β(rθ(x,y)−z0))if y is desirableλU σ ⁣(β(z0−rθ(x,y)))if y is undesirablev(x,y)= \begin{cases} \lambda_D\,\sigma\!\left(\beta(r_{\theta}(x,y)-z_0)\right) & \text{if } y \text{ is desirable} \\ \lambda_U\,\sigma\!\left(\beta(z_0-r_{\theta}(x,y))\right) & \text{if } y \text{ is undesirable} \end{cases}v(x,y)={λD​σ(β(rθ​(x,y)−z0​))λU​σ(β(z0​−rθ​(x,y)))​if y is desirableif y is undesirable​

The logistic surrogate σ\sigmaσ is doing a lot of work here. It keeps the optimization stable, and it gives the loss the familiar S-shape where improvements matter most near the reference point and then saturate.

The final training objective is just the expectation of the label-specific loss over the data:

LKTO(πθ,πref)=Ex,y∼D[λy−v(x,y)]L_{\text{KTO}}(\pi_{\theta},\pi_{\text{ref}})=\mathbb{E}_{x,y\sim D}[\lambda_y-v(x,y)]LKTO​(πθ​,πref​)=Ex,y∼D​[λy​−v(x,y)]

The important thing is not the exact algebra. It is the behavior:

  • desirable outputs are pushed up relative to the reference,
  • undesirable outputs are pushed down,
  • the two directions can be weighted differently,
  • and the reference point keeps the update from becoming a blunt “push harder forever” objective.

Why the reference point matters

The reference point is what keeps KTO from degenerating into a simple classifier. If the model can raise a desirable example’s score only by increasing the KL divergence everywhere, the loss stops being useful. The method wants a more specific improvement: make the chosen output better without letting the model drift uncontrollably.

That is why the paper uses a KL-based reference point, but does not backpropagate through it. The reference point is there to control where the value function saturates, not to become another learned target.

In practice, estimating that reference point exactly would be expensive because it would require sampling from the current policy. So the paper uses a biased microbatch estimate instead. Outputs are shifted within the batch to create mismatched pairs, and those are used to estimate a shared z0z_0z0​.

This is an efficiency tradeoff, not a theoretical ideal. The authors explicitly accept the bias because the lower variance and the extra practicality make the training loop usable. That also matches the broader story of the paper: the loss shape matters enough that a slightly biased but stable estimate can still work well.

One subtle point is that the estimate is often less important after SFT. If the reference model is already close to the desirable data, the KL term can become small enough that setting the reference-point estimate to zero is sometimes reasonable. When KTO is used without SFT, though, the estimate becomes more important.

How binary preference data becomes KTO data

KTO is built for binary desirability labels, but the paper also shows how to apply it to standard preference datasets. The simplest conversion is:

  • treat the preferred response as desirable,
  • treat the rejected response as undesirable.

That is intentionally naive. The paper is candid that a richer decomposition of preferences into binary feedback could do better. Still, even this simple transformation is enough to test the core idea.

The more interesting case is when the data is already naturally unpaired. KTO can train on one output per prompt, with no need to preserve the pairwise structure. That is one reason the method is appealing: it does not need preference pairs as a hard requirement.

The picture to keep in mind is that KTO is comfortable in imbalanced data regimes. The separate λD\lambda_DλD​ and λU\lambda_UλU​ weights let the loss compensate for uneven class counts, so the method can still learn even when desirable outputs are scarce.

The hyperparameters are part of the method

KTO is sensitive to tuning, and the paper is explicit about that.

The learning rate matters most. The recommended rate is often much higher than DPO’s because the reference-adjusted reward is smaller in magnitude. In practice, the paper finds that KTO often wants something like 5×10−65\times10^{-6}5×10−6 rather than the smaller DPO default.

The β\betaβ parameter controls how fast the logistic value saturates. Lower values are better when the model already had SFT, while higher values can help when KTO is used more directly as the alignment stage.

The loss weights λD\lambda_DλD​ and λU\lambda_UλU​ handle class imbalance. The paper’s rule of thumb is to keep the effective desirable-to-undesirable weighting in a reasonable range after accounting for class counts. That matters because the objective is not trying to be symmetric in a naive way; it is trying to reflect the data balance and the practical importance of making good outputs more likely.

This is one reason KTO is not a “set it and forget it” method. The loss shape is powerful, but only if the scale and weighting make sense for the data.

What the method looks like when it works

The main experimental result is simple: KTO is competitive with or better than DPO across the reported model scales.

What matters in the figure is not the exact winrate, but the pattern. KTO is not just matching DPO at one size. It stays competitive from small to large models, and for the Llama family it is strong enough that KTO alone can match the SFT+DPO setup.

The paper also shows a second important behavior: at sufficient scale, KTO does not need SFT first.

The visual lesson here is that skipping SFT can destabilize some alignment methods. DPO without SFT can wander into rambling or hallucinated conversation structure, while KTO is less prone to that failure in the reported setup. That suggests the KTO loss is not just cheaper in data format; it can also be more forgiving about where it sits in the post-training pipeline.

Why the paper thinks KTO works

The authors give two broad explanations.

First, preference likelihood and human utility are not the same thing. Two reward functions can live in the same equivalence class and still induce different human value distributions. So optimizing a preference model does not necessarily mean optimizing the best human-aligned utility.

Second, KTO behaves like a selective updater. If an example is too easy or too hard under the current implied reward, the gradient shrinks. That means very noisy or very difficult examples can get ignored rather than overfit. In a messy human-feedback setting, that can be a feature, not a bug.

That selective behavior cuts both ways. It can protect the model from noise, but it can also underfit complex data if the learning rate and weighting are not tuned well. The paper is honest about that tradeoff.

There is also an interesting worst-case result in the paper’s analysis: when preferences are contradictory, DPO can end up favoring the minority-preferred outcome under some conditions, while KTO with balanced weights is biased toward the majority-preferred one. That does not make KTO universally better, but it does explain why the method can behave more robustly when the data is noisy or intransitive.

When to use KTO instead of DPO

The paper’s practical advice is fairly direct.

Use KTO when your feedback is naturally binary, or when your preference data is imbalanced, noisy, or only partially trustworthy. In those settings, the binary loss and the reference-point shape are a good match.

Use DPO when your preferences are clean, consistent, and well-structured, and when you want the pairwise formulation itself.

That is the real takeaway: KTO is not “DPO but simpler.” It is a different inductive bias. If the dataset fits that bias, KTO can be very strong. If the dataset is clean enough that pairwise preferences are reliable, DPO may still be the right tool.

What to keep in mind

KTO’s mental model is compact:

an output is compared to a KL-based baseline, the loss rewards movement in the right direction, and good versus bad examples are weighted separately.

If you remember only one thing, remember this: KTO is trying to preserve the useful part of preference optimization without depending on preference pairs. It does that by making the loss itself more human-like, with a reference point, asymmetry, and saturation.

That is why the paper matters. It shows that alignment quality is not only about the data format or the model size. The shape of the objective can be just as important.
