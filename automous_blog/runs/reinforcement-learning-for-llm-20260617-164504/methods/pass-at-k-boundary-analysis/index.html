Does Reinforcement Learning Really Incentivize Reasoning Capacity in LLMs Beyond the Base Model?

This paper asks a deceptively simple question: when reinforcement learning with verifiable rewards improves a reasoning model, is it actually teaching the model new reasoning, or is it mostly making the model better at sampling reasoning it already knew how to do?

That distinction matters. A model can look much better at pass@1 simply because the correct answer becomes more likely on the first try. But that does not necessarily mean the model can solve a wider set of problems. To test the difference, the paper switches from “did the first sample work?” to “does any sample work if I keep trying?” That is the whole point of pass@k.

The big idea is easiest to remember as a boundary test:

RLVR may sharpen the model’s search around familiar good answers, but does it actually move the edge of what the model can solve?

Why pass@k changes the question

Most evaluation habits collapse reasoning into a single number. That is fine if you only care about average-case performance, but it is too coarse if you want to know whether a model has latent ability hiding behind unlucky decoding.

pass@k makes the evaluation more generous. For each problem, the model gets k independent samples. If any one of them is correct, the problem counts as solved. That matters because a model with a strong internal reasoning path may fail at k=1 yet succeed once you give it more tries.

The paper uses that lens to compare three things:

  • base models,
  • RLVR-trained models,
  • distilled models.

The comparison is not just about which one scores highest at k=1. It is about which one owns a wider part of the reasoning space as sampling increases.

What to notice here is the shape of the curves. RLVR looks good early, when you only allow one or a few samples. But the base model keeps gaining ground as sampling gets broader. In other words, RLVR is winning the first-shot race while losing the wider-search race.

The black box: sample more, then measure the boundary

The method itself is not complicated. The paper’s logic is:

  1. sample many outputs from a model for each problem,
  2. compute pass@k across tasks,
  3. compare how the curves behave for base, RLVR, and distilled models,
  4. inspect the actual reasoning traces and their likelihood under the base model.

That last step is important. If RLVR really invented new reasoning paths, then those paths should look qualitatively different from what the base model already tends to produce. If they are just reorganizing probability mass inside the base distribution, the evidence should show up in the sampling and perplexity patterns.

So the paper is doing two things at once:

  • measuring how many problems are solvable,
  • asking where the successful trajectories came from.

That combination is what turns the paper from a benchmark report into a boundary analysis.

What happens on math, code, and vision

Across all three task families, the same pattern appears.

At small k, RLVR-trained models usually look stronger. That part is unsurprising. RL directly optimizes for correct final answers, so it should raise the chance that one sampled response is right.

The surprising part comes later. As k grows, the base model often catches up and then surpasses the RLVR model. The gap appears not only on one benchmark or one model size, but repeatedly across the paper’s comparisons.

The interpretation is subtle but important: RLVR is improving sampling efficiency on problems the base model already had some chance of solving, while often narrowing the set of problems that remain reachable under broader sampling.

So the model gets better at landing on a known good path quickly, but not necessarily better at opening new paths.

The hidden story inside the accuracy histogram

The paper makes this more concrete by looking at the accuracy distribution before and after RLVR training.

Here is the important pattern to notice. RLVR increases the mass near accuracy 1.0, which is exactly what you would expect if it is making correct answers more likely. But it also increases the mass at accuracy 0, which means more problems become effectively unsolved.

That looks odd at first, but it matches the larger pass@k result. RLVR is not uniformly making the model better across the whole problem set. It is concentrating probability on a subset of solvable problems while leaving, or even hurting, the rest.

This is why the paper argues that pass@1 is not enough. If you only look at the average first attempt, you miss the fact that the training signal may be trading breadth for concentration.

The key claim: the base model already contains the useful paths

The paper then asks the most important follow-up question: are the successful RLVR generations actually new, or were they already latent in the base model?

To probe that, it looks at accuracy coverage and perplexity. The coverage analysis shows that many problems solved by RLVR were already solvable by the base model, while the reverse case is rare. The rough set relationship is almost one-way: the RLVR-solvable problems look close to a subset of the base model’s solvable problems.

Then the perplexity analysis sharpens the point.

What to notice here is that responses generated by the RLVR model tend to sit in a region of the base model’s distribution that already had low perplexity. In plain language, the base model already considered those answers fairly natural.

That is the heart of the paper’s argument.

RLVR is not discovering an entirely new reasoning space. It is mostly increasing the probability of a region the base model already knew how to reach.

You can think of it like this: if the base model is a map of all the roads it already knows, RLVR is not adding new roads. It is putting more traffic on the roads that were already there.

Why distillation behaves differently

The paper contrasts RLVR with distillation because the two methods can look similar from a distance but do very different things.

Distillation takes long reasoning traces from a stronger teacher and trains a smaller student on them. That means the student can inherit reasoning patterns that were not originally prominent in its own base distribution.

This figure is the cleanest side-by-side contrast in the paper. The distilled model sits above the base model across the pass@k curve, not just at k=1. That matters because it suggests broader capability transfer, not just better sampling of existing capability.

So the paper’s message is not “RL never helps.” RL clearly helps the first-sample success rate. The point is narrower and sharper: distillation is more reliable at moving the reasoning boundary, while RLVR is better understood as a way to reweight what was already in reach.

Why the RL algorithm choice matters less than you might think

One of the paper’s strongest practical observations is that different RL variants do not change the story very much.

The algorithms vary a bit in immediate efficiency, but the sampling-efficiency gap stays large. That means the gap between RLVR and the base model is not just a quirk of one optimizer. It looks like a structural problem.

The paper’s interpretation is straightforward: these algorithms are all operating inside the same constrained exploration regime. If the search space is huge and the reward is binary, the optimizer mostly learns to favor trajectories that already sit near the base model’s prior.

That explains why changing the RL recipe can slightly shift the curve, but usually does not erase the large-sample gap.

Training makes the distribution sharper, not wider

The paper also tracks what happens as RL training continues.

The short version is: pass@1 improves, but the large-k boundary gets worse.

That looks like sharpening rather than expansion. The model becomes more confident on its favored paths, but less diverse overall. The paper also checks entropy, and the result is consistent with that story: lower entropy contributes to the narrowing, but does not fully explain it on its own.

In other words, the model is not just getting “smarter.” It is getting more peaked.

That distinction is easy to miss, but it is the central mechanism behind the paper’s interpretation. A more peaked distribution can be a better sampler for a familiar answer, while still being a worse explorer of the problem space.

Why the boundary still looks base-model-limited even for stronger models

The paper also checks a more capable, near-frontier reasoning model and finds the same qualitative pattern.

At low k, RLVR helps. At higher k, the advantage shrinks or disappears.

That is important because it suggests the phenomenon is not just a weakness of small models. The paper’s claim is broader: current RLVR recipes seem to improve efficiency inside the existing reasoning boundary more than they expand the boundary itself.

So scaling the model alone does not automatically fix the issue. The open question is whether much larger RL budgets, richer exploration, or different training signals can eventually change the picture.

Where the paper thinks the bottleneck comes from

The discussion section gives a useful mental model for why this happens.

Traditional RL successes, like game playing, often start from scratch in a controlled action space. Reasoning LLMs do the opposite. They start with a pretrained prior in an enormous language space, and RL only gets to nudge that prior.

That prior is a strength and a constraint at the same time. It gives the model useful structure, but it also makes exploration hard. If you stray too far, you get nonsense. If the reward is sparse and binary, the safest move is usually to keep leaning on the paths the model already knows.

That makes the paper’s conclusion feel inevitable in hindsight: binary outcome rewards are good at reweighting existing routes, but weak at discovering brand-new ones.

What would have to change

The paper does not stop at criticism. It points to the kinds of changes that might actually expand the boundary:

  • better exploration in a higher-level abstraction space,
  • curriculum-style data that builds up easier subskills first,
  • finer-grained process rewards instead of only final-answer reward,
  • multi-turn agentic training with feedback from tools or environments.

These are not cosmetic tweaks. They are attempts to change the geometry of the search problem itself. If the action space is too large and the reward too sparse, then ordinary token-level RL will mostly polish paths that already exist.

What to remember

The easiest way to remember this paper is:

  • pass@1 tells you how often the model hits the answer quickly.
  • pass@k tells you how wide the model’s reasoning boundary really is.
  • RLVR usually improves the first number more than the second.
  • Distillation is more likely to expand the second number.

That is the paper’s main contribution. It separates “sampling efficiency” from “capability expansion” and shows that current RLVR mostly lives in the first bucket.

If you want the shortest possible summary, it is this: current RLVR often makes a base model more likely to choose a good path, but not much more likely to know new paths exist.
