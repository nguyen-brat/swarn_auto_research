Recursive Rubric Decomposition (RRD): Recursive Rubric Decomposition for LLM Judges and Reward Modeling

RRD starts from a very practical complaint: if you ask an LLM to judge open-ended responses with one pass of loosely written rubrics, the result is often messy. The rubric list may be too broad to separate good answers from merely decent ones, too overlapping to add much new information, or even pointed in the wrong direction. And once those same rubrics become reward signals for reinforcement fine-tuning, the noise gets amplified instead of averaged away.

The paper’s answer is not to abandon rubrics, but to treat them more like a diagnostic system. Start with coarse checks, split the checks that are too blunt, throw away the ones that point the wrong way, and then down-weight criteria that are basically saying the same thing. The result is a sharper rubric set that is easier for a judge to use and more stable as a reward source.

The whole idea in one picture

Before diving into the mechanics, it helps to see the pipeline as a whole.

What to notice is that RRD is not just “generate rubrics and stop.” It is a loop. First the model proposes rubrics from the prompt and sample responses. Then it keeps refining the coarse pieces that are too vague. Then it filters out bad or redundant items. Finally, it decides how to combine the remaining criteria so repeated ideas do not get overcounted.

That is the core design move: quality comes from structure, not from a single giant rubric list.

Why a one-pass rubric list is brittle

In open-ended judging, the problem is not that there are no useful criteria. The problem is that useful criteria are usually entangled.

A response can be fluent but incomplete, detailed but off-target, safe but evasive, or logically sound but poorly organized. If a rubric just says “be helpful” or “be complete,” it is too broad to tell which response is actually better. If it says “mention the key facts,” it may overlap with half a dozen neighboring checks. And if it accidentally rewards the weaker side of a comparison, the whole judgment can go sideways.

That is why the paper treats rubric quality as a first-class problem. The target is not merely coverage. The target is a rubric set that is:

  • informative enough to separate responses,
  • comprehensive enough to cover the important dimensions,
  • and non-redundant enough that the same signal is not counted three times.

RRD is the mechanism that tries to enforce all three at once.

Start coarse, then sharpen what is too broad

The first step is deliberately simple. The model proposes an initial set of rubrics from the prompt and a handful of sampled responses. Those samples matter because they ground the rubric proposal in concrete differences instead of abstract wording. If one response is verbose, another is concise, and a third misses the main point, the rubric generator has something real to work with.

From there, RRD asks a useful question: does this rubric actually separate the candidates?

If a rubric is satisfied by too many responses, it is usually too broad. In diagnosis terms, it is like a test that comes back positive for almost everyone. That test may be related to the condition, but it is not specific enough to be useful on its own. RRD recursively decomposes that rubric into finer subcriteria until the new pieces are more discriminative.

This is the part that makes the method feel less like flat prompt engineering and more like structured reasoning. Instead of assuming the first rubric draft is good enough, the method treats broad criteria as signals that there is more structure hiding underneath.

The paper’s intuition is straightforward: if many responses satisfy the same rubric, that rubric has probably collapsed several different ideas into one. Split it apart until the parts start saying something distinct.

Filtering is as important as expanding

Decomposition alone would not be enough. If you keep splitting forever, you can end up with a brittle pile of hyper-specific criteria that track accidental details in the sampled outputs rather than stable preference dimensions.

So RRD adds two kinds of filtering.

The first is directionality filtering. Some rubrics look plausible but consistently favor the wrong side of the comparison. The paper uses a conservative sanity check based on whether the rubric would systematically prefer a weaker reference model over a stronger one. If it does, that rubric is likely misaligned and should be discarded.

The second is redundancy filtering. Even after decomposition, different rubrics can still be almost paraphrases of one another. Keeping all of them would make the final score look more confident than it really is, because the same idea would be counted multiple times under slightly different wording.

This is where the medical analogy in the paper is especially helpful. A physician does not keep every correlated symptom as separate evidence. Elevated CRP and ESR both point toward inflammation, but they are not independent pieces of truth. RRD uses the same kind of discipline: keep the useful signals, but do not double-count the ones that are really saying the same thing.

Why correlation changes the weighting problem

Once the rubric set is cleaned up, the next question is how to combine the surviving rubrics.

The paper’s key observation is that correlation is not just a filtering issue. Even if every rubric is individually useful, a group of highly correlated rubrics can distort the final aggregation because they effectively vote as a bloc. In plain terms, three nearly identical rubrics should not count like three independent pieces of evidence.

That is why RRD does not stop at “remove duplicates.” It also asks how to weight the remaining rubrics in a way that respects their covariance structure.

For open-ended tasks, the paper argues that the preference signal is often distributed across many dimensions rather than concentrated in one dominant rubric. In that setting, heuristic weighting is fragile. A cleaner approach is to move into a whitened space, where the correlated directions are normalized away, and then use uniform weighting there.

You do not need the algebra to understand the motivation. The point is simple: if two rubrics are almost the same, the aggregator should not treat them like two independent votes. Whitening is the mechanism that makes that correction systematic instead of ad hoc.

What whitening is doing, in plain language

Think of the rubric set as a coordinate system for quality.

If the coordinates are tangled together, then a score built from them will overreact to directions that happen to be repeated. Whitening untangles the coordinates first. After that, the remaining dimensions are closer to being independent, so averaging them is less misleading.

That is why the paper prefers whitened uniform weighting for open-ended tasks. It is a way of saying: “first remove the repeated signal, then average the distinct signal.” The result is a judgment rule that is less likely to be dominated by a cluster of near-duplicates.

The paper also notes that heuristic weights can still make sense when there is one clearly dominant rubric. But for open-ended settings, where the evidence is spread out, correlation-aware weighting is the safer choice.

From sharper rubrics to better judgments

The first test of the method is whether the refined rubrics actually help an LLM judge rank responses better.

What to notice here is not just the top-line score. The important pattern is that naive rubric generation can actually hurt performance, while recursive refinement plus correlation-aware weighting recovers and improves it. That tells you the method is not merely adding complexity for its own sake. It is removing a real failure mode: broad or misaligned rubrics can confuse the judge more than they help.

The best-performing variant is the one that combines decomposition with whitened weighting. That lines up with the paper’s theory. Once the rubric set contains many valid but correlated criteria, the aggregation step matters just as much as the generation step.

Why this matters even more for reward modeling

Judging is only half the story. The more interesting question is what happens when the rubric-based judge becomes the reward source for reinforcement fine-tuning.

This is where noisy rubrics can become expensive. A small bias in a one-off verdict is annoying. The same bias, repeated over thousands of training updates, can shape the policy in the wrong direction. So the reward source needs to be stable, discriminative, and calibrated enough to support optimization.

RRD is trying to give the policy trainer exactly that kind of signal.

The shape of the curves is the interesting part. RRD produces stronger and steadier reward gains than earlier rubric baselines, which suggests that the refined rubric set is not just more accurate in a single comparison but also more usable as a learning signal across many updates. The paper’s message is that a better rubric is not only a better explanation tool. It is a better optimization target.

How the reward signal reaches the policy

The downstream training setup is conceptually simple.

The policy samples responses to prompts from a natural chat distribution. A rubric-based judge scores those responses. Those scores become the reward signal for reinforcement fine-tuning. If the judge is brittle, the policy learns brittle behavior. If the judge is informative and stable, the policy gets a cleaner training signal.

That is why RRD pays so much attention to the judge itself. In open-ended alignment, the reward model is not an auxiliary detail. It is the thing that shapes the policy.

The paper then checks whether this better reward source actually transfers to useful behavior outside the training loop.

What matters here is the transfer story. Better rubric structure improves not just the reward curves during training, but also the final policy quality on both broad assistant tasks and a high-stakes domain like health. That is a strong sign that the method is improving the supervision signal rather than just gaming one benchmark.

Why the recursive loop adapts to task difficulty

One subtle strength of RRD is that the number of rubrics is not fixed up front.

That is a good fit for open-ended tasks. Some prompts are simple and only need a few useful checks. Others are subtle and need a much finer breakdown before the competing responses become separable. RRD grows until the rubric space stops yielding novel, valid, non-redundant ideas.

This makes the method feel task-adaptive rather than rigid. A simple prompt does not need a huge rubric tree. A complex prompt should not be forced to stop at a shallow one. Recursive decomposition gives the method a way to expand only as far as needed.

The tradeoff is obvious: if the recursion goes too far, the rubric set can fragment into overly specific checks. The paper treats that as a real failure mode, not a theoretical edge case.

Where the method can break

RRD is powerful, but it is not free of assumptions.

Directionality filtering can be brittle, especially in value-sensitive domains. A rubric that does not favor the stronger reference model is not automatically wrong, because some tasks legitimately reward caution, brevity, or refusal. The paper treats this filter as a conservative sanity check, not a universal truth test.

Recursive decomposition can also over-fragment the rubric set if it is pushed too far. Once that happens, the criteria stop reflecting stable preference dimensions and start tracking sample-specific quirks.

And whitening depends on a reasonable estimate of covariance. If the rubric statistics are poor, the weighting step can be less reliable. So the correlation-aware part of the method is principled, but not magical.

The thing to remember

RRD is easiest to remember as a three-part discipline for rubrics:

  1. start with coarse criteria grounded in the prompt and sample responses,
  2. split any criterion that is too broad to separate the candidates,
  3. remove misaligned or redundant items, then combine the rest in a way that does not overcount correlated signals.

That is the whole method.

The reason it matters is that open-ended judging and reward modeling fail in a very specific way: the rubric set becomes noisy before the model ever sees a clean training signal. RRD tries to fix the rubric set itself, so the judge is sharper and the reward is more trustworthy. In that sense, the paper is not just about better prompts for evaluation. It is about turning rubrics into structured supervision that can survive both judgment and training.
