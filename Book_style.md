# Skill: Practical Technical Handbook Writing Style

## Goal
Write clear, practical, example-driven technical handbook content that helps readers understand, apply, and evaluate a method.

## Style
Use a calm, instructional tone. Write like an expert teacher, not a research paper. Prefer plain language first, then formal details. Avoid hype. Be useful, critical, and structured.

## Chapter Pattern
For each topic or method, follow this order:

1. **Short definition**
   - Explain what the method is in 2–4 sentences.

2. **Motivation**
   - Explain why the method exists.
   - State the problem it solves.

3. **Intuition**
   - Use a simple mental model, analogy, or small example.

4. **Formal explanation**
   - Add math, algorithm steps, or technical detail only after intuition.

5. **Worked example**
   - Show how the method works on a realistic use case.
   - Explain what the result means.

6. **Interpretation**
   - Teach the reader how to read outputs, plots, scores, or decisions.

7. **Strengths**
   - State when the method works well.

8. **Limitations**
   - State failure cases, assumptions, risks, and misleading uses.

9. **Practical guidance**
   - Explain when to use it, when not to use it, and what alternatives exist.

10. **Tools or implementation notes**
   - Mention useful libraries, APIs, or implementation concerns if relevant.

## Writing Rules
- Start simple; become technical gradually.
- Prefer concrete examples over abstract explanation.
- Compare methods when useful.
- Include warnings and edge cases.
- Be honest about uncertainty and limitations.
- Use headings, short paragraphs, lists, and diagrams/tables when helpful.
- Do not overclaim.
- Do not write like marketing material.
- Do not only describe; help the reader decide and apply.
